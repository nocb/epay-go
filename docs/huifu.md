# 汇付天下（斗拱平台）接入说明

## 1. 异步回调流转机制

在 Epay-go 中，汇付天下的支付成功回调是**全自动化**流转的，您**不需要**在汇付商户后台手动配置固定通知地址。

### 处理流程
1. **自动生成回调地址**：当下游商户（如 `new-api`）发起支付请求时，`epay-go` 在请求汇付接口时会自动把本系统的回调地址传给汇付。
   - 支付宝通道传给汇付的回调地址形如：`http(s)://您的易支付域名/api/pay/notify/hf-alipay`
   - 微信通道传给汇付的回调地址形如：`http(s)://您的易支付域名/api/pay/notify/hf-wxpay`
2. **汇付异步通知**：用户在支付宝/微信完成支付后，汇付会向上述地址发送支付成功的异步请求。
3. **验签与状态更新**：`epay-go` 的 `HandleNotify` 收到通知后，会自动完成 RSA 验签，并将数据库中的订单状态修改为 `已支付`。
4. **下游商户通知**：状态更新后，`epay-go` 会自动读取商户最初下单时传入的 `notify_url`，并以易支付的标准签名格式发送回调通知给下游（如 `new-api`），通知其业务成功（例如给用户增加配额）。

### 注意事项
- 请确保 `epay-go` 系统部署在外网，并且外界可以正常访问其域名或公网 IP。
- 商户密钥和汇付公钥配置无误，否则会导致回调验签失败，订单状态无法更新。
